# English Learning MVP Design

## Purpose

Build a local single-machine MVP for English vocabulary and phrase learning. The first version validates the full teacher-to-student loop: teachers import learning material, the system completes missing learning fields, students study and test themselves, exams are generated randomly, and results can be reviewed later.

## Scope

The MVP uses built-in teacher and student entry points instead of real account registration and login. The UI keeps the role split visible so a later version can add authentication and permissions without changing the core workflow.

In scope:

- Teacher import from Word, text-based PDF, and pasted text.
- Teacher preview and confirmation before terms are saved.
- Group management by grade and chapter.
- Student learning by grade or chapter.
- Self-test with immediate feedback.
- Random exam generation with saved results.
- Exam result list and detail review with correct/wrong filtering.
- Responsive Pad and H5 layouts.
- Automated tests for import, enrichment, quiz generation, result saving, and responsive behavior.

Out of scope for the MVP:

- Real teacher/student login.
- OCR for scanned PDFs.
- Multi-school or multi-class management.
- Payment, deployment operations, and production monitoring.
- Human-recorded audio.

## Recommended Approach

Use a Next.js full-stack monolith with SQLite. Pages, server actions or API routes, import parsing, enrichment, quiz generation, and result saving live in one project. This keeps the MVP simple while preserving a clean path to real accounts and a hosted database later.

## User Roles

### Teacher

The teacher imports source material, assigns it to a grade or chapter, reviews parsed and generated fields, confirms the import, and manages the term library.

### Student

The student selects a grade or chapter, studies paginated cards, runs self-tests, generates exams, submits answers, and reviews previous results.

## Groups

The system creates default grade groups such as:

- 1年级上册
- 1年级下册
- 2年级上册
- 2年级下册

Chapters are child groups under grade groups. A term can belong to one or more groups if the same word or phrase appears in multiple books or chapters.

## Term Types

The system distinguishes words from phrases or fixed expressions.

### Word

Words may contain:

- Original text
- Phonetic symbol
- Part of speech
- Chinese meaning
- Example sentence
- Explanation

### Phrase

Phrases and fixed expressions do not need phonetic symbols or parts of speech. They contain:

- Original phrase
- Chinese meaning
- Example sentence
- Common usage context

Example:

```text
look after
中文：照顾
例句：She looks after her little brother after school.
常用场景：用于表达照顾人、动物或物品，常见于家庭、学校和日常生活场景。
```

## Teacher Import Flow

The import flow supports `.docx`, text-based `.pdf`, and pasted text. Text-based means the PDF text can be selected and copied; scanned image PDFs are not handled in the MVP.

Flow:

1. Teacher selects grade and optional chapter.
2. Teacher uploads a Word/PDF file or pastes text.
3. The system extracts candidate entries.
4. The parser tries to identify term text, phonetic symbol, part of speech, Chinese meaning, example sentence, group, and chapter.
5. Existing fields from the teacher's material are preserved.
6. Missing fields are sent to enrichment.
7. The teacher reviews a preview table.
8. The teacher edits any incorrect rows.
9. The teacher confirms import.
10. The system saves new terms or merges duplicates.

Important rule: imported fields are authoritative. AI or mock enrichment only fills missing fields and must not overwrite teacher-provided content.

## Enrichment Flow

The enrichment module uses a provider interface:

- Real AI provider when an API key is configured.
- Mock provider when no API key exists or the real provider fails.

For words, enrichment may fill:

- Phonetic symbol
- Part of speech
- Chinese meaning
- Example sentence
- Explanation

For phrases, enrichment may fill:

- Chinese meaning
- Example sentence
- Common usage context

The provider returns structured JSON. The server validates the JSON before it reaches the preview page. If validation fails, the affected row falls back to mock enrichment and is marked as mock-generated.

Each field records its source:

- `parsed`: extracted from teacher material.
- `ai_generated`: generated by the AI provider.
- `mock_generated`: generated by the fallback provider.
- `edited`: edited manually by the teacher in the preview page.

## Duplicate Handling

Terms are normalized before matching:

- Trim leading and trailing spaces.
- Lowercase Latin letters.
- Collapse repeated internal whitespace.
- Preserve the original display text.

Rules:

- Duplicate words or phrases do not create new term rows.
- New Chinese meanings are appended to the existing term.
- Repeated meanings are ignored.
- Teacher-provided existing fields are not overwritten by generated fields.
- If the same meaning has a new example sentence, the system keeps it as an additional example for that meaning when possible.

## Data Model

### `groups`

Stores grades and chapters.

Fields:

- `id`
- `name`
- `parent_id`
- `sort_order`
- `created_at`
- `updated_at`

### `terms`

Stores the word or phrase identity.

Fields:

- `id`
- `text`
- `normalized_text`
- `term_type`: `word` or `phrase`
- `created_source`
- `created_at`
- `updated_at`

### `term_groups`

Connects terms to grades and chapters.

Fields:

- `term_id`
- `group_id`

### `term_meanings`

Stores meanings and learning content.

Fields:

- `id`
- `term_id`
- `part_of_speech`
- `chinese_meaning`
- `example_sentence`
- `explanation`
- `usage_context`
- `field_sources_json`
- `created_at`
- `updated_at`

For phrases, `part_of_speech` is empty.

### `import_batches`

Stores each import attempt.

Fields:

- `id`
- `source_type`
- `file_name`
- `target_group_id`
- `status`
- `created_at`
- `confirmed_at`

### `import_rows`

Stores parsed preview rows before confirmation.

Fields:

- `id`
- `batch_id`
- `row_index`
- `raw_text`
- `parsed_json`
- `enriched_json`
- `duplicate_status`
- `validation_status`
- `created_at`

### `exam_sessions`

Stores each submitted exam.

Fields:

- `id`
- `selected_group_ids_json`
- `question_count`
- `score`
- `correct_count`
- `started_at`
- `submitted_at`

### `exam_answers`

Stores a full answer snapshot.

Fields:

- `id`
- `exam_session_id`
- `question_index`
- `question_type`
- `prompt`
- `options_json`
- `correct_option_id`
- `selected_option_id`
- `is_correct`
- `explanation`
- `term_snapshot_json`

The snapshot makes result review stable even if the teacher edits the term library later.

## Student Learning Flow

The student chooses a grade or chapter. The system loads terms and meanings for that group.

Display content:

- Words show text, phonetic symbol, browser speech button, part of speech, Chinese meanings, example sentences, and explanations.
- Phrases show phrase text, Chinese meanings, example sentences, and common usage contexts.

Pagination:

- Pad shows several cards per page with a visible group sidebar.
- H5 shows one card per page or a compact fixed number of cards per page.
- Study pages are designed to avoid body-level vertical scrolling on H5.

## Self-Test Flow

The student chooses a grade or chapter. The system randomly selects one term meaning per question.

Question types:

- Word or phrase to Chinese meaning.
- Sentence blank to phrase, for phrase questions when a good example sentence exists.

Rules:

- ABCD has exactly one correct answer.
- Correct answer position is randomized every time.
- For words with multiple meanings or parts of speech, each test can select a different meaning.
- Wrong answers show the correct answer and explanation.
- For phrases, the explanation includes common usage context.

## Exam Flow

The student selects one or more grades or chapters and starts an exam. Each start creates a new random paper.

Rules:

- Default exam length is 100 questions.
- If the library has enough distinct meanings, avoid exact repeats.
- If the library is small, allow repeated terms when they test different meanings.
- If fewer than 100 usable questions exist even after meaning-level expansion, generate the available number and show a clear notice.
- Students submit once and receive a score.
- The system stores the full question and answer snapshot.

## Result Review Flow

The result list shows previous exam sessions with date, selected group names, score, correct count, and total count.

The result detail page supports filters:

- All
- Correct
- Wrong

Each item shows:

- Prompt
- Student answer
- Correct answer
- Explanation
- Term or phrase usage context

## Responsive Design

The MVP supports Pad and H5 from the start.

Pad:

- Two-column workspace.
- Left side keeps grade and chapter navigation visible.
- Right side shows teacher tables, learning cards, tests, exams, or result details.

H5:

- Single-page flow.
- Bottom navigation for Learn, Test, Exam, and Results.
- Group selection opens as a full-screen picker.
- Learning and self-test pages use fixed-height paged content and avoid body vertical scrolling.
- Exam and result detail pages may scroll because 100 questions and wrong-answer review can be long.

## Page Structure

Teacher pages:

- `/teacher/import`
- `/teacher/import/[batchId]/preview`
- `/teacher/library`

Student pages:

- `/learn`
- `/self-test`
- `/exam`
- `/results`
- `/results/[sessionId]`

API routes or server actions:

- Import file or pasted text.
- Enrich missing fields.
- Confirm import batch.
- List groups and terms.
- Generate self-test questions.
- Generate exam papers.
- Submit exam answers.
- Read result sessions and details.

## Technical Architecture

Use:

- Next.js App Router
- TypeScript
- SQLite
- Prisma
- `mammoth` for `.docx` parsing
- `pdf-parse` or an equivalent package for text-based PDF parsing
- A provider-based enrichment module
- Browser Web Speech API for MVP audio playback
- Vitest for unit tests
- Playwright for end-to-end and responsive tests

Suggested module boundaries:

- `src/lib/import/`: document and pasted-text parsing.
- `src/lib/enrichment/`: AI and mock enrichment providers.
- `src/lib/terms/`: normalization, validation, duplicate detection, and merging.
- `src/lib/quiz/`: self-test and exam question generation.
- `src/lib/results/`: score calculation and answer snapshots.
- `src/app/teacher/`: teacher UI.
- `src/app/learn/`: learning UI.
- `src/app/self-test/`: self-test UI.
- `src/app/exam/`: exam UI.
- `src/app/results/`: result list and review UI.
- `tests/unit/`: pure business logic tests.
- `tests/e2e/`: Playwright browser tests.

## Error Handling

- File parsing fails: show a clear message and offer pasted-text import.
- AI provider fails: use mock enrichment and mark generated fields as mock-generated.
- Parsed row is ambiguous: keep it in preview with validation status requiring teacher edit.
- Term library is empty: student pages show an empty state that points to teacher import.
- Exam question pool is too small: generate the available question count and show the count before the exam starts.
- Browser speech is unavailable: hide or disable the speech button with a short unavailable state.

## Automated Test Coverage

Unit tests:

- Parse pasted text with existing word, phonetic symbol, part of speech, Chinese meaning, and sentence.
- Preserve imported fields and fill only missing fields.
- Ensure phrases do not receive phonetic symbols or parts of speech.
- Normalize and deduplicate repeated words and phrases.
- Merge new meanings and ignore repeated meanings.
- Generate ABCD options with exactly one correct answer.
- Randomize correct option placement.
- Select different meanings for multi-meaning words.
- Calculate exam scores.
- Save answer snapshots independent of later term edits.

End-to-end tests:

- Teacher imports pasted text and confirms preview.
- Teacher imports a `.docx` sample with existing fields.
- AI failure falls back to mock enrichment.
- Student studies a selected group.
- Student answers a self-test question and sees immediate feedback.
- Student submits an exam and sees a score.
- Result detail filters all, correct, and wrong answers.
- Pad viewport shows sidebar navigation.
- H5 viewport shows bottom navigation.
- H5 learning and self-test pages do not use body vertical scrolling.

## Acceptance Criteria

- A teacher can import text-based learning material, review generated rows, and save terms.
- Teacher-provided fields are never overwritten by generated content.
- Missing word fields are completed by AI or mock fallback.
- Missing phrase fields are completed with Chinese meaning, example sentence, and usage context only.
- Duplicate uploads merge instead of creating duplicate term records.
- Students can learn, self-test, take an exam, and review results.
- Exams are randomized on each start and save full answer snapshots.
- Pad and H5 layouts follow their respective navigation and pagination patterns.
- Automated tests cover import, enrichment, duplicate merge, quiz generation, exam results, and responsive behavior.
